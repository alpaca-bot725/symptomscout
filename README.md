# SymptomScout

A mobile-first symptom triage web app built for a college health-education
project. **This is not a medical device** — it is an educational demo of how
rules-based triage systems work.

> This tool is for education only and does not provide medical diagnoses.
> Always consult a healthcare professional.

## Run it

```bash
npm install
npm run dev        # development server
npm run build      # production build (enables the PWA service worker)
npm run preview    # serve the production build locally
```

Built with React 19, Vite 8, and Tailwind CSS 4.

## How it works

```
User flow:   Safety check → Body region → Symptoms → Follow-ups → Results
                   │
                   └── any global red flag → Emergency screen (results skipped)
```

### Architecture — data and logic are separated on purpose

| Piece | File | Role |
|---|---|---|
| Knowledge base | `src/data/conditions.json` | 30 conditions, weighted symptoms, red flags, body regions. **Edit this to expand the app — no code changes needed.** |
| Triage engine | `src/engine/triage.js` | Pure functions that interpret the JSON. Heavily commented. |
| Episode history | `src/lib/history.js` | localStorage persistence + doctor-summary export. |
| UI | `src/components/` | One component per screen. |

### The scoring algorithm (see `src/engine/triage.js` for details)

1. **Red flags first.** Eight global red flags (chest pain, breathing
   difficulty, stroke signs, severe bleeding, suicidal thoughts, thunderclap
   headache, stiff neck + fever, throat swelling) hard-override everything
   and show emergency guidance instead of results.
2. **Derived flags.** Follow-up answers become synthetic symptoms
   (e.g. fever = "high" → `fever_over_104`) so JSON rules can react to them.
3. **Weighted scoring.** Each condition symptom has a weight: 3 = hallmark,
   2 = common, 1 = supporting. `percent = matched weights / total weights`.
4. **Matching rule.** A condition only appears if a hallmark symptom matched
   OR percent ≥ 40% — one vague symptom can't surface everything.
5. **Escalation only goes up.** Condition red flags, severity ≥ 8/10, and
   age (<2 or 65+) can raise urgency but never lower it, and only explicit
   red flags may declare an emergency.

Urgency levels: Self-care → See a doctor within a few days → Urgent care
today → Call 911.

### Adding a condition

Append an object to `conditions` in `src/data/conditions.json` following the
`_schema` notes at the top of that file. New symptom ids are picked up
automatically; assign them to a region in `body_regions` (or let them fall
into "General / Whole Body").

### PWA

`public/manifest.webmanifest` + `public/sw.js` make the app installable to a
home screen and fully offline-capable (all data is local). Icons are
generated by `node scripts/generate-icons.mjs` using only Node built-ins.

### Privacy

Everything stays on the device: no network calls, no accounts, no analytics.
History lives in localStorage and can be deleted in-app.
